## Введение

**PM Assistant** — это система для автоматизации управления проектами и встречами с использованием искусственного интеллекта. Система автоматически транскрибирует аудиозаписи встреч, создает резюме, извлекает задачи и помогает проект-менеджерам эффективно управлять проектами.

### Основные возможности

- 🎤 **Автоматическая транскрибация** аудиозаписей встреч
- 📝 **AI-суммаризация** транскриптов встреч
- ✅ **Автоматическое извлечение задач** (Action Items)
- 📊 **Управление проектами** и участниками
- 🔔 **Система уведомлений** о статусе обработки
- 🔐 **Интеграция с Active Directory** для авторизации
- 📁 **Хранение аудио** в объектном хранилище MinIO S3

## Быстрый старт

### Предварительные требования

- Docker и Docker Compose (версия 2.0+)
- MinIO S3 сервер (внешний) для хранения аудио файлов
- Gemini API ключ для AI-обработки
- Active Directory сервер (для авторизации)

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone <repository-url>
cd pm-assistant

# 2. Создать .env файл
cp env.example .env

# 3. Настроить переменные окружения в .env:
#    - S3_ENDPOINT_URL, S3_ACCESS_KEY, S3_SECRET_KEY
#    - GEMINI_API_KEY
#    - LDAP_SERVER, LDAP_BASE_DN и другие настройки AD

# 4. Запустить все сервисы
make up

# ИЛИ вручную:
docker compose up -d
sleep 5
docker compose exec api alembic upgrade head
```

### Проверка работы

- **API документация**: http://localhost:8000/docs
- **Health check**: http://localhost:8000/health
- **Логи**: `docker compose logs -f`

## Бизнес-документация

### Назначение системы

PM Assistant предназначен для автоматизации рутинных задач проект-менеджеров:

1. **Обработка встреч** — автоматическая транскрибация и анализ аудиозаписей
2. **Управление проектами** — организация встреч по проектам, управление участниками
3. **Извлечение задач** — автоматическое создание Action Items из транскриптов
4. **Документирование** — создание структурированных заметок и резюме встреч

### Целевая аудитория

#### 1. Проект-менеджеры (PM)
- Создают и управляют проектами
- Организуют встречи
- Просматривают транскрипты и резюме
- Управляют задачами (Action Items)
- Имеют полный доступ к проектам

#### 2. Менеджеры (Manager)
- Участвуют в проектах
- Организуют встречи в рамках проектов
- Просматривают результаты обработки
- Управляют задачами

#### 3. Участники (Member)
- Участвуют в проектах
- Просматривают встречи и результаты обработки
- Имеют ограниченный доступ

### Основные функции

#### Управление проектами

- **Создание проектов** с описанием и метаданными
- **Управление доступом** — добавление/удаление участников с разными ролями
- **Архивирование** проектов без удаления данных
- **Интеграция с Jira и Confluence** — структура для будущей интеграции
- **Счетчики** — автоматический подсчет участников и встреч

#### Управление встречами

- **Создание встреч** с загрузкой аудио файлов
- **Привязка к проектам** или создание некатегоризированных встреч
- **Перемещение встреч** между проектами
- **Поиск и фильтрация**:
  - По названию
  - По проекту
  - По организатору
  - По периоду (дата начала/конца)
  - По длительности
- **Сортировка**:
  - По дате (новые→старые, старые→новые)
  - По длительности (короткие→длинные, длинные→короткие)
- **Управление заметками** (Notes)
- **Управление задачами** (Action Items)
- **Получение временных ссылок** на аудио файлы

#### AI-обработка встреч

- **Автоматическая транскрибация** через Whisper API
- **Суммаризация транскриптов** через Gemini AI
- **Извлечение Action Items** из транскрипта
- **Асинхронная обработка** через очередь задач (Redis + ARQ)
- **Отслеживание статуса** обработки (прогресс, этапы)
- **Повторная обработка** при ошибках

#### Система уведомлений

- **Автоматические уведомления** о статусе обработки встреч
- **Центр уведомлений** для просмотра всех уведомлений
- **Отметка прочитанных** уведомлений
- **Счетчик непрочитанных** уведомлений
- **Удаление уведомлений**

### Бизнес-процессы

#### Процесс обработки встречи

1. **Создание встречи**
   - Пользователь создает встречу и загружает аудио файл
   - Аудио сохраняется в MinIO S3
   - Создается запись в базе данных

2. **Запуск обработки**
   - Пользователь запускает обработку встречи
   - Задача добавляется в очередь Redis
   - Создается уведомление о начале обработки

3. **Транскрибация**
   - Worker получает задачу из очереди
   - Аудио транскрибируется через Whisper API
   - Транскрипт сохраняется в базе данных
   - Статус обновляется

4. **Суммаризация**
   - Транскрипт отправляется в Gemini AI для суммаризации
   - Резюме сохраняется в базе данных
   - Статус обновляется

5. **Извлечение задач**
   - Из транскрипта извлекаются Action Items через Gemini AI
   - Задачи сохраняются в базе данных
   - Статус обновляется на "завершено"

6. **Уведомление**
   - Создается уведомление о завершении обработки
   - Пользователь получает уведомление в центре уведомлений

#### Процесс управления проектом

1. **Создание проекта**
   - PM создает проект с названием и описанием
   - Опционально указывает данные для интеграции с Jira/Confluence
   - PM автоматически добавляется как участник с ролью PM

2. **Управление доступом**
   - PM добавляет участников с ролями (PM/Manager/Member)
   - Участники получают доступ к проекту согласно ролям
   - PM может удалять участников

3. **Организация встреч**
   - Участники проекта создают встречи и привязывают их к проекту
   - Встречи отображаются в списке встреч проекта
   - Счетчик встреч автоматически обновляется

4. **Архивирование**
   - PM может архивировать проект
   - Архивные проекты скрыты из основного списка
   - Данные сохраняются, проект можно разархивировать

### Интеграции

#### Текущие интеграции

- **Active Directory (LDAP)** — авторизация пользователей
- **MinIO S3** — хранение аудио файлов
- **Whisper API** — транскрибация аудио
- **Gemini AI** — суммаризация и извлечение задач
- **Redis** — очередь задач для асинхронной обработки

#### Планируемые интеграции

- **Jira** — автоматическое создание задач из Action Items
- **Confluence** — публикация резюме встреч в Confluence
- **AI чат** — интерактивный чат для создания заданий

### Метрики и аналитика

Система предоставляет следующие метрики:

- Количество проектов (активных/архивных)
- Количество встреч по проектам
- Количество участников по проектам
- Статус обработки встреч
- Количество Action Items по проектам

### API Endpoints

#### Пользователи (`/api/users/`)
- `POST /login` — вход через Active Directory
- `POST /logout` — выход
- `GET /me` — текущий пользователь
- `GET /` — список пользователей

#### Проекты (`/api/projects/`)
- `GET /` — список проектов
- `POST /` — создание проекта
- `GET /{id}` — получение проекта
- `PUT /{id}` — обновление проекта
- `DELETE /{id}` — удаление проекта
- `POST /{id}/archive` — архивирование проекта
- `POST /{id}/unarchive` — разархивирование проекта
- `POST /{id}/access` — добавление участника
- `DELETE /{id}/access/{user_id}` — удаление участника

#### Встречи (`/api/meetings/`)
- `GET /` — список встреч (с фильтрацией и сортировкой)
- `POST /` — создание встречи с аудио
- `GET /{id}` — получение встречи
- `PUT /{id}` — обновление встречи
- `DELETE /{id}` — удаление встречи
- `POST /{id}/process` — запуск обработки
- `GET /{id}/processing-status` — статус обработки
- `GET /{id}/transcript` — получение транскрипта
- `GET /{id}/summary` — получение резюме
- `GET /{id}/action-items` — получение задач
- `POST /{id}/notes` — добавление заметок
- `GET /{id}/audio-url` — временная ссылка на аудио

#### Уведомления (`/api/notifications/`)
- `GET /` — список уведомлений
- `GET /unread-count` — количество непрочитанных
- `PUT /{id}/read` — отметить как прочитанное
- `DELETE /{id}` — удалить уведомление

Полная документация API доступна по адресу: `http://localhost:8000/docs`

### Переменные окружения

Основные переменные окружения (см. `env.example`):

#### База данных
- `DATABASE_URL` — строка подключения к PostgreSQL
- `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`

#### Приложение
- `APP_NAME` — название приложения
- `DEBUG` — режим отладки (true/false)
- `API_PREFIX` — префикс API (по умолчанию `/api`)

#### Active Directory / LDAP
- `LDAP_SERVER` — адрес LDAP сервера
- `LDAP_BASE_DN` — базовый DN
- `LDAP_USER_DN` — домен пользователей
- `LDAP_SERVICE_USER` — сервисный пользователь
- `LDAP_SERVICE_PASSWORD` — пароль сервисного пользователя
- `LDAP_ADMIN_GROUP` — группа администраторов

#### S3 / MinIO
- `S3_ENDPOINT_URL` — URL MinIO сервера
- `S3_ACCESS_KEY` — ключ доступа
- `S3_SECRET_KEY` — секретный ключ
- `S3_BUCKET_NAME` — имя bucket (по умолчанию `pm-assistant`)

#### AI сервисы
- `GEMINI_API_KEY` — API ключ Gemini
- `GEMINI_MODEL` — модель Gemini (по умолчанию `gemini-1.5-pro`)
- `WHISPER_SERVER_URL` — URL локального Whisper сервера
- `USE_LOCAL_WHISPER` — использовать локальный Whisper (true/false)

#### Redis
- `REDIS_URL` — URL Redis сервера

- **Схемы** (`schemas.py`) — Pydantic схемы для валидации
- **Маршруты** (`routes.py`) — FastAPI endpoints
- **Сервисы** (`services.py`) — бизнес-логика
- **Селекторы** (`selectors.py`) — запросы к БД

#### Тестирование

```bash
# Запустить тесты
make test

# ИЛИ
uv run pytest tests/ -v
```

#### Форматирование кода

```bash
# Форматировать код
make format

# Проверить форматирование
make lint
```
